## 搬瓦工换机房的前提：先搞清楚这个功能怎么用

搬瓦工（BandwagonHost）的机房迁移功能叫 **Migrate to another DC**，藏在 KiwiVM 控制面板的左侧菜单里。

**正常迁移的操作步骤：**

1. 登录搬瓦工账户，进入 Services → My Services
2. 找到你的 VPS，点 KiwiVM Control Panel 进入后台
3. 左侧菜单找到 **Migrate to another DC**
4. 右侧会列出当前套餐可用的机房列表，选一个目标机房
5. 点 **Confirm on next step**，再点 **Start Migration** 开始迁移
6. 等邮件通知——通常几分钟到十几分钟，数据量大的话可能更久

整个过程数据不会丢失，迁移期间 VPS 大部分时间是正常运行的，只在最后几秒到几分钟会短暂关机。迁完之后 IP 地址会变，如果你绑了域名记得改解析。

说说正常情况。操作完是这样的。

## 换不了机房？先对着报错信息找原因

### 情况一：报错 733102（Plan does not support migrations）

提示：`Plan does not support migrations (733102)`

这个最直接。意思就是你买的套餐本身不支持迁移机房，不是你操作有问题，是套餐级别不行。

主要发生在两种场景：

- 买了很早期的老套餐，当年有些特价方案明确锁定只能用默认机房，续费了这么多年还在用的那种
- 当前套餐的级别没有你想去的那个机房的权限（比如普通 KVM 套餐是进不了 CN2 GIA 机房的）

低级别套餐可以迁入同级机房，CN2 GIA-E 这类高端套餐可以迁到低级别机房，但反过来不行。想用 DC9 CN2 GIA 机房，手里得是支持这个机房的套餐才行。

**解决方法：** 如果当前套餐用着还凑合，就继续续着。如果真的想换更好的机房，只有重新买支持目标机房的套餐。

### 情况二：报错 734104 / 734105（Region is full）

提示：`Region is full. Please try again in a few hours. (734104)` 或 `New location has no available IP addresses... (734105)`

字面意思——目标机房暂时满了，没有空余 IP 可以分配给你。

这种情况通常是临时的。等一两个小时，等有人迁出去腾出 IP，你就能进去了。不是你的问题，也不是套餐问题。

**解决方法：** 等。一般等个几小时再试，或者换个其他机房先用着，隔段时间再切回来。不需要提工单，系统资源自然会释放。

### 情况三：报错 734151 / 3009005（系统暂时不可用）

提示：
- `Migration system is currently not available. Please try again in a few minutes. (734151)`
- `MAINTENANCE: KiwiVM control panel is temporarily unavailable. Please try again in a few minutes. (3009005)`

搬瓦工的迁移后端暂时出问题了，或者同时迁移的人太多把系统压住了。

这个也是临时状态，概率不大。

**解决方法：** 等 15-30 分钟再试，一般就好了。实在不行提工单告诉客服。

### 情况四：报错 734152（Migration backend is currently not available）

**提示：** `Migration backend is currently not available for this VPS. Please try again in 10-15 minutes. (734152)`

这是最让人头疼的一个。它表面上说"等 10-15 分钟再试"，但等了几个小时、换了好几个目标机房，还是这个错——那基本上是 IP 被封了。

IP 被封之后，搬瓦工的控制面板后端没法和你的 VPS 建立有效连接，迁移命令发不下去，就一直报这个错。

**怎么确认 IP 是不是真的被封：**

登录 KiwiVM 控制面板，找到你的 VM ID（首页有显示），然后在浏览器地址栏访问：


https://kiwivm.64clouds.com/你的VM_ID/main-exec.php?mode=blacklistcheck


把"你的VM_ID"替换成那串数字，打开后点 Test，看结果：
- 显示 **IP NOT BLOCKED** → IP 正常，可能是系统临时问题，多试几次或提工单
- 显示 **IP BLOCKED** → 确认被封，下一步换 IP

IP 被封状态下，迁移机房是不可能成功的，不用再试了。

**解决方法：** 确认 IP 被封之后，唯一出路是买新 IP。免费换 IP 的时代已经过去了，目前搬瓦工付费换 IP 是人工操作的，申请后最多等 24 小时。换完新 IP 之后，迁移功能就恢复正常了。

## 换机房需要注意的几个细节

**IP 会变。** 迁移到新机房后，IP 地址会重新分配。如果 VPS 上跑着网站、绑了域名解析，迁完要第一时间去改 DNS 记录，不然服务会断。

**迁回原机房 IP 不一定会变。** 先从 A 迁到 B，再从 B 迁回 A，如果原来在 A 的 IP 还没被别人占用，那 IP 还是之前那个。快速迁出去再迁回来，概率上能保住原来的 IP。

**迁移占流量。** 迁移过程中传输的数据会计入当月流量配额，不过通常量很小，可以忽略不计。

**数据不会丢。** 整个迁移过程中数据是保留的，只在最后阶段会短暂停机几秒到几分钟。

## 常见问题

**Q：搬瓦工换机房免费吗？**  
A：机房迁移本身免费，没有次数限制。但如果 IP 被封需要换 IP，这个是付费的，价格以申请页面为准。

**Q：换机房之后数据会丢失吗？**  
A：不会。迁移过程中所有数据都会保留，只在最后阶段短暂停机。迁移前可以先创建一个快照备份，以防万一。

**Q：为什么我在 KiwiVM 后台看不到 Migrate to another DC 这个选项？**  
A：如果这个菜单项根本不显示，或者点进去直接提示 Plan does not support migrations，说明当前套餐不支持机房迁移功能。只能换套餐或继续用现在的机房。

**Q：同一个 IP 反复被封，是不是套餐本身的问题？**  
A：不是套餐问题，是使用方式的问题。IP 被封是因为该 IP 在国内被检测到并列入黑名单，和套餐类型没有关系。换 IP 后注意使用规范，避免再次被封。

**Q：换机房后多久能用？**  
A：通常 10-15 分钟内完成，数据量大的 VPS 可能要更久。完成后会收到邮件通知，按新的 IP 和端口连接即可。
